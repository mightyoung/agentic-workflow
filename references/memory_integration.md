# 记忆系统 (v4.5)

> 借鉴 proactive-agent-3.1.0 的 WAL 协议 + 三层记忆架构
> 本地文件实现，无需外部插件依赖

---

## 核心原则

> **Chat history is a BUFFER, not storage.**
> **SESSION-STATE.md is your RAM.** — 工作内存，会话结束后可选提炼到长期记忆

---

## 三层记忆架构

```
┌─────────────────────────────────────────────────────────────┐
│  Layer 1: SESSION-STATE.md (工作内存)                        │
│  - 当前任务的所有细节                                        │
│  - 每条消息都更新关键信息                                    │
│  - 会话结束时提炼到 Layer 2/3                               │
├─────────────────────────────────────────────────────────────┤
│  Layer 2: memory/YYYY-MM-DD.md (每日日志)                   │
│  - 会话期间的所有活动记录                                    │
│  - 每日一个文件，可追溯历史                                  │
│  - 定期从中提炼到 Layer 3                                    │
├─────────────────────────────────────────────────────────────┤
│  Layer 3: MEMORY.md (长期记忆)                              │
│  - 核心经验和模式提炼                                        │
│  - 持久化，可跨会话使用                                      │
└─────────────────────────────────────────────────────────────┘
```

## WAL 协议 (Write-Ahead Logging)

### 核心思想

每条用户消息都先检查是否包含需要记忆的信息，如有则先写 SESSION-STATE.md，再响应用户。

### 触发条件

| 类型 | 示例 | 处理 |
|------|------|------|
| **修正信息** | "是X，不是Y" / "其实..." / "错了" | 立即更新SESSION-STATE |
| **专有名词** | 人名、地名、公司名、产品名 | 记录到专有名词表 |
| **偏好** | "我喜欢X" / "不喜欢Y方式" | 记录偏好表 |
| **决策** | "用X方案" / "选择Y" | 记录决策表 |
| **具体数值** | 数字、日期、ID、URL | 记录数值表 |

### 执行流程

```
用户消息
    ↓
扫描所有触发条件
    ↓
有匹配？
    ├── 否 → 直接响应
    └── 是 → STOP，不开始写响应
              ↓
         WRITE → 更新 SESSION-STATE.md
              ↓
         THEN → 响应用户
```

### WAL 触发模式

```python
WAL_TRIGGERS = [
    # 修正信息
    { pattern: /(?:不是|其实|错了)/, type: 'correction' },
    # 偏好
    { pattern: /(?:我喜欢?|我不喜欢?)/, type: 'preference' },
    # 决策
    { pattern: /(?:用|选择|决定)/, type: 'decision' },
    # 专有名词（需要人工判断）
]
```

## 记忆操作命令

| 命令 | 功能 |
|------|------|
| `/check-memory` | 检查当前 SESSION-STATE 状态 |
| `/save-memory` | 将会话提炼到 MEMORY.md |
| `/search-memory [关键词]` | 搜索历史记忆 |
| `/clear-session` | 清除 SESSION-STATE，开始新会话 |
| `/show-daily [日期]` | 查看指定日期的每日日志 |

## 与 MCP 记忆的关系

**可选增强**：如果已配置 `MCP claude-mem` 插件，可以额外使用：

```python
# 任务开始前检索
memory_search(query=当前任务描述, namespace=agentic-workflow)

# 任务完成后存储
memory_store(
    key=任务名,
    value=经验总结,
    namespace=agentic-workflow,
    tags=["经验", "模式"]
)
```

**重要**：MCP 是可选的增强，**不依赖 MCP 也能正常工作**。

## 最佳实践

1. **每会话创建 SESSION-STATE.md** - 会话开始时创建
2. **WAL 协议扫描** - 每条消息检查触发条件
3. **主动提炼** - 会话结束或有意义时提炼到 MEMORY.md
4. **搜索历史** - 新任务开始时搜索相关经验

---

## 注意事项

- SESSION-STATE.md 是**必须**的，其他层级是**可选**
- WAL 协议是**轻量级**的，不需要外部插件
- 定期清理过时的每日日志
- 长期记忆需要**手动提炼**，不会自动创建
