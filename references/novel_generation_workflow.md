## 一、核心机制

### 1.1 四合一同步生成

生成小说章节时，**必须同步生成**以下四件套：

| 类型 | 输出位置 | 说明 |
|------|----------|------|
| **章节正文** | `chapters/` | 小说章节内容 |
| **播客剧本** | `podcasts/` | 有声书剧本，3-4章合并 |
| **视频分镜** | `video_prompts/` | Seedance 2.0 视频提示词 |
| **视觉参考** | `visual_reference/` | 角色+场景描述 |

**核心目的**：视觉参考（角色+场景）是多类型创作的**一致性基石**，确保：
- 角色在小说、播客、视频中形象统一
- 场景在不同内容形式中描述一致
- 视频分镜与视觉参考一一对应

### 1.2 一一对应机制

视频Prompt中的每个场景必须与视觉参考文件**一一对应**：

```
视频Prompt引用                    视觉参考文件
──────────────────────────────────────────────────
视频分镜中的场景ID  ──────────→  visual_reference/scenes/v{卷号}_ch{章节号}_{场景名}.md
视频Prompt中的人物ID  ───────→  visual_reference/characters/v{卷号}_{角色名}.md
```

**命名规则**：
- 场景文件：`v{卷号}_ch{章节号}_{场景名}.md`
- 角色文件：`v{卷号}_{角色名}.md`

## 二、字数标准

### 2.1 章节字数

| 标准 | 要求 | 用途 |
|------|------|------|
| **最少** | 3500字（不含空格/换行） | 确保播客约15分钟 |
| **推荐** | 3500-4000字 | 最佳播客时长 |
| **最多** | 8000字 | 避免单集过长 |

### 2.2 字数计算规则

```
有效字数 = 总字符数 - 空格 - 换行 - 标点符号（可选）
```

**验证方法**：使用脚本统计纯中文字符数

turnaround sheet, multiple angles view,
front view, side view, 3/4 view, back view,
# 写实光线
cinematic studio lighting, soft shadows,
skin texture detail, fabric texture detail,
# 画布控制
centered on canvas, [肩部以下-full body] composition,
no background, no props, no watermark,
# 输出规格
high resolution, ultra detailed, 8K UHD
```

## AI绘图提示词

### 正向提示词（写实风格·短剧场景）

场景图必须使用**写实风格**，并符合**短剧视频**的视觉语言：

```
# 写实基础
photorealistic, hyperrealistic, 8K, masterpiece, best quality,
[场景描述],
[建筑风格], [环境布置],
[光线描述], [氛围描述],
[季节], [时间段],
# 写实细节
highly detailed, architectural photography,
realistic lighting, volumetric lighting,
atmospheric perspective, depth of field,
# 短剧风格（适合视频分镜）
cinematic, dramatic lighting,
short drama aesthetic, web series style,
close-up friendly composition,
# 画布控制
wide angle shot, establishing shot,
medium shot, close-up compatible,
centered composition, [宽高比] aspect ratio,
# 输出规格
high resolution, ultra detailed, 8K UHD,
cinematic color grading, film grain
```

### 短剧风格说明

| 元素 | 要求 | 说明 |
|------|------|------|
| 光线 | dramatic lighting, cinematic | 短剧强调戏剧冲突，光线要有明暗对比 |
| 构图 | close-up friendly | 便于后期切近景/特写 |
| 氛围 | emotional atmosphere | 场景需传递情绪，辅助叙事 |
| 色彩 | cinematic color grading | 统一色调，符合短剧审美 |

### 参考形象

```
{中文场景描述}
风格：写实（photorealistic）+ 短剧（short drama）
光线：{时间段} {光质}
氛围：{氛围描述}
```

| 元素 | 要求 | 说明 |

## 五、工作流程

### 5.1 五合一同步生成

生成小说时，**必须同步生成**以下五件套：

| 类型 | 输出位置 | 说明 |
|------|----------|------|
| **封面素材** | `cover/` | AI绘图提示词（每卷1套） |
| **章节正文** | `chapters/` | 小说章节内容 |
| **播客剧本** | `podcasts/` | 有声书剧本，3-4章合并 |
| **视频分镜** | `video_prompts/` | Seedance 2.0 视频提示词 |
| **视觉参考** | `visual_reference/` | 角色+场景描述 |

**生成顺序**：
1. **先封面**：确定整本小说的视觉基调
2. **先角色**：确保人物形象一致
3. **再场景**：基于角色活动场景
4. **后章节**：章节内容引用已创建的视觉参考
5. **播客/视频**：基于章节内容生成

### 5.2 批次生成（每3-4章）

```
[PLANNING] 规划批次
    │
    ▼
[EXECUTING] 同步生成五件套
    │
    ├── 生成封面素材 (每卷首次生成)
    ├── 生成章节正文 (ch001-ch004)
    ├── 生成播客剧本 (3-4章合并)
    ├── 生成视频分镜 (场景拆分)
    └── 生成视觉参考 (角色+场景)
    │
    ▼
[REVIEWING] 质量检查
    │
    ├── 字数验证 (≥3500字/章)
    ├── 一一对应验证
    └── 内容一致性验证
    │
    ▼
[COMPLETE] 输出报告
```

## 六、多格式一致性保障

### 6.1 一致性核心原则

视觉参考（角色+场景）是**唯一的真实来源（Single Source of Truth）**：

```
小说文本  ───引用───→ 视觉参考（角色+场景）
                              │
           ┌─────────────────┼─────────────────┐
           ▼                 ▼                 ▼
      播客剧本          视频分镜           AI绘图
    （引用角色/场景）  （引用视觉参考）   （引用视觉参考）
```

### 6.2 Agent协作模式

| Agent | 职责 | 输入 | 输出 |
|-------|------|------|------|
| **主角Agent** | 生成章节正文 | 大纲+视觉参考 | 章节内容 |
| **播客Agent** | 生成有声书剧本 | 章节内容 | 播客剧本 |
| **视频Agent** | 生成视频分镜 | 视觉参考 | 视频Prompt |
| **绘图Agent** | 生成视觉参考 | 角色/场景描述 | AI绘图Prompt |

### 6.3 一致性检查清单

生成内容时必须验证：

- [ ] 角色外貌特征与视觉参考一致
- [ ] 场景描述与视觉参考一致
- [ ] 视频分镜引用的视觉参考ID存在
- [ ] 播客剧本中的场景描述与章节一致

## 八、一致性保障机制（参考 ComfyUI，Seedance 2.0 为主）

> **重要说明**：视频生成使用 **Seedance 2.0**。ComfyUI 仅作为参考架构和一致性机制的启发来源。
>
> ComfyUI 的 PhotoMaker、ControlNet、LoRA 等机制为小说工作流提供了**一致性保障的设计思路**，但具体实现以 Seedance 2.0 的最佳实践为准。

### 8.1 一致性层级

| 层级 | ComfyUI 机制 | 小说工作流实现 | 作用 |
|------|-------------|---------------|------|
| **角色身份** | PhotoMaker ID Encoder | 角色视觉参考（多角度） | 确保同一角色在不同场景中外观一致 |
| **场景结构** | ControlNet | 场景视觉参考 | 确保场景布局、环境元素一致 |
| **风格氛围** | LoRA | AI绘图提示词（风格标签） | 确保整体视觉风格统一 |

### 8.2 PhotoMaker 模式（角色身份一致性）

ComfyUI 的 PhotoMaker 使用参考图像编码角色身份：

```
参考图像 → CLIP Vision Encoder → ID embeddings → 与文本Prompt融合
                      ↓
          特殊token "photomaker" 标记角色注入位置
```

**对应到小说工作流**：

```
角色视觉参考 → 角色描述模板 → 角色ID编码 → 融入各类型内容
                                              ↓
                    ┌──────────┬──────────┬──────────┐
                    ↓          ↓          ↓          ↓
               章节正文    播客剧本   视频Prompt  AI绘图Prompt
```

### 8.3 ControlNet 模式（场景结构一致性）

ComfyUI 的 ControlNet 提供结构/姿态控制：

- **姿态控制**：保持角色姿态一致性
- **边缘检测**：保持构图一致性
- **深度感知**：保持空间关系一致

**对应到小说工作流**：

```
场景视觉参考 → 场景结构描述 → ControlNet conditioning → 场景一致性
                              ↓
        ┌─────────────────────┼─────────────────────┐
        ↓                     ↓                     ↓
   场景正面图             场景俯视图            场景细节图
```

### 8.4 LoRA 模式（风格一致性）

ComfyUI 的 LoRA（Low-Rank Adaptation）用于轻量级风格适配：

- 可训练的轻量级权重调整
- 可组合多个 LoRA 实现复合风格
- 支持风格迁移

**对应到小说工作流**：

```
角色LoRA → 基于角色图像训练 → 角色专属风格适配器
场景LoRA → 基于场景图像训练 → 场景专属风格适配器
风格LoRA → 基于整体风格训练 → 统一风格适配器
```

### 8.5 Agent 协作模式（参考 AI Workflow Agent）

参考 ComfyUI AI Workflow Agent 的架构：

```
┌─────────────────────────────────────────────────────────────┐
│                    小说创作 Agent 编排                        │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐   │
│  │ 主角Agent │→ │播客Agent │→ │视频Agent │→ │绘图Agent │   │
│  └──────────┘  └──────────┘  └──────────┘  └──────────┘   │
└─────────────────────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────────────────────┐
│                      记忆系统                                │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐              │
│  │ 短期记忆  │  │ 中期记忆  │  │ 长期记忆  │              │
│  │ (当前章节)│  │ (卷内一致)│  │ (跨卷一致)│              │
│  └──────────┘  └──────────┘  └──────────┘              │
