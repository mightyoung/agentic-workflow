# 子智能体派生成模块

当需要并行执行独立任务时，使用此模块派生子智能体。

## 何时派生子智能体

| 场景 | 子智能体 | 执行模式 |
|------|----------|----------|
| 多文件审查 | reviewer | 并行 |
| 多模块开发 | coder | 并行 |
| 技术调研 + 规划 | researcher + planner | 并行 |
| 搜索 + 实现 | researcher + coder | 串行 |

## 派生流程

```
1. 识别可并行任务
2. 选择合适的子智能体
3. 派生子智能体
4. 收集结果
5. 汇总给主智能体
```

## 决策矩阵

### 任务依赖分析

```
独立任务 → 并行执行
  ↓
依赖任务 → 串行执行
  ↓
混合任务 → 分组并行
```

### 子智能体选择

| 任务类型 | 子智能体 | 备选 |
|----------|----------|------|
| 搜索调研 | researcher | - |
| 任务规划 | planner | - |
| 代码实现 | coder | ecc-workflow /tdd |
| 代码审查 | reviewer | ecc-workflow /code-review |
| Bug调试 | debugger | - |

## 调用示例

### 示例1：并行代码审查

```
用户: 帮我审查这个10个文件的PR

→ 分析：10个文件，可并行审查
→ 派生 3 个 reviewer 子智能体
  - reviewer_1: 文件 1-4
  - reviewer_2: 文件 5-7
  - reviewer_3: 文件 8-10
→ 并行执行
→ 汇总审查结果
→ 返回给用户
```

### 示例2：调研+规划并行

```
用户: 我想开发一个电商网站

→ 分析：需要调研+规划
→ 并行派生：
  - researcher: 搜索电商最佳实践
  - planner: 规划任务拆分
→ 等待两个结果
→ 汇总给主智能体
```

### 示例3：搜索后实现

```
用户: 帮我实现分布式事务

→ 分析：需要先调研再实现
→ 派生 researcher: 搜索最佳实践
→ 等待搜索结果
→ 派生 coder: 基于调研结果实现
→ 返回实现结果
```

## 限制

- 最大并发子智能体数：5
- 单个子智能体超时：5分钟
- 超出限制时使用队列或拒绝
