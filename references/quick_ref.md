# 快速参考卡

> 核心路由关键词速查

## 路由关键词

| 模块 | 触发关键词 | 不触发关键词 |
|------|------------|--------------|
| RESEARCH | 最佳实践, 有什么, 有哪些, 选型, 部署, 方法 | - |
| THINKING | 谁最懂, 专家, 顶级, best minds, 分析, 怎么做, 理解 | - |
| PLANNING | 计划, 规划, 拆分, 设计, 安排, 制定 | - |
| DEBUGGING | bug, 错误, 调试, 修复, 报错, 崩溃, 异常, 定位, Error | - |
| REVIEWING | 审查, review, 检查, 代码审查, pr审查 | - |
| EXECUTING | 开发, 实现, 写, 创建 | 系统, 架构, 完整 |
| EXECUTING-FAST | 写一个, 创建个, 帮我写, 生成一个 | 系统, 架构, 支付 |

## 不触发关键词

天气, 笑话, 你好, hi, hello, bye, 谢谢, ok, yes, no, maybe

## 核心铁律

1. **穷尽一切** - 没尝试3种方案前不说"无法解决"
2. **先做后问** - 先搜索/读源码，再提问
3. **主动出击** - 检查同类问题，补充合理需求

## 文件模板

- `task_plan.md` - 任务计划
- `findings.md` - 研究发现
- `progress.md` - 进度追踪

## 压力升级

| 失败次数 | 等级 | 动作 |
|---------|------|-----|
| 2次 | L1 | 切换方案 |
| 3次 | L2 | 搜索+假设 |
| 4次 | L3 | 7项检查 |
| 5次+ | L4 | PoC+隔离 |

## 问题分级

- 🔴 致命：崩溃、数据丢失、安全漏洞
- 🟡 严重：功能错误、性能下降、API破坏
- 🟢 建议：风格、可读性、优化
