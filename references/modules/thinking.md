# 思考模块详解 (THINKING)

> 专家模拟思维 + 链式推理 + 反例检验

## 核心原则

不要问"你怎么看"，而是问"这个问题谁最懂？TA会怎么说？"。

## 链式推理步骤（必做）

1. **定义问题**：用一句话说清楚要解决什么
2. **拆解要素**：这个问题由哪些子问题组成？
3. **验证前提**：每个前提是否为真？
4. **推导链条**：A→B→C，每步必须有充分理由
5. **寻找反例**：有没有推翻这个结论的例子？
6. **得出结论**：用一句话总结，必须包含条件

## 专家模拟步骤（可选）

当需要外部视角时：
1. 谁最可能解决过这个问题？
2. TA 会怎么思考？
3. 引用 TA 的公开言论

### 专家识别模式

| 问题领域 | 可能的专家 |
|----------|-----------|
| 系统架构 | 系统架构师、技术负责人 |
| 算法问题 | 算法工程师、学术研究者 |
| 安全问题 | 安全专家、渗透测试工程师 |
| 性能优化 | 性能工程师、SRE |
| 产品设计 | 产品经理、UX设计师 |

## 输出格式

```markdown
## 思考分析

### 问题定义
[一句话]

### 要素拆解
- 要素1：xxx
- 要素2：xxx

### 推理链条
1. 因为 [前提A]
2. 所以 [中间结论B]
3. 因此 [最终结论C]

### 反例检验
- 反例1：[描述] → [反驳/成立]
- 反例2：[描述] → [反驳/成立]

### 结论
[带条件的结论]
```

## 推理检查清单

- [ ] 问题定义是否清晰无歧义？
- [ ] 要素拆解是否完整？
- [ ] 每个前提是否经过验证？
- [ ] 推导链条每步是否有充分理由？
- [ ] 是否尝试寻找反例？
- [ ] 结论是否包含条件？
