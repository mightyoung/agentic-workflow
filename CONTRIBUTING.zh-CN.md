# 贡献指南

感谢您有兴趣参与贡献！

## 行为准则

参与本项目即表示您同意维护一个尊重和包容的环境。

## 如何贡献

### 报告 Bug

1. 检查现有 issues 以避免重复
2. 使用 bug 报告模板
3. 包含：
   - 清晰的描述
   - 复现步骤
   - 预期与实际行为
   - 环境详情

### 功能建议

1. 检查现有的 issues/discussions
2. 使用功能请求模板
3. 说明：
   - 您要解决的问题
   - 提出的解决方案
   - 考虑过的替代方案

### 拉取请求

1. Fork 仓库
2. 创建功能分支：`git checkout -b feature/my-feature`
3. 进行更改
4. 运行测试：`npm test`
5. 使用清晰的消息提交
6. 推送并创建 PR

## 开发环境设置

```bash
# 克隆你的 fork
git clone https://github.com/YOUR_USERNAME/integrated-dev-workflow.git
cd integrated-dev-workflow

# 安装依赖
npm install

# 运行测试
npm test
```

## 风格指南

- 遵循现有文件结构
- 使用清晰、描述性的提交消息
- 对任何更改更新文档
- 为新功能添加测试

## 认可

贡献者将列在 README 和发布说明中。
