## 三、设计改进建议

### 3.1 路由逻辑优化

以下建议属于当时的优化方向；当前仓库已将真实路由逻辑对齐到 `scripts/router.py` 和 `tests/run_phase_test.py`。

根据测试结果，路由逻辑工作正常，但可做以下增强：

**建议1: 增加意图强度检测**

```python
# 当前: 简单关键词匹配
if 包含("最佳实践"):
    → RESEARCH

# 建议: 增加意图强度
if 包含("最佳实践") and 包含("怎么做"):
    → RESEARCH + 强制搜索
elif 包含("最佳实践"):
    → RESEARCH + 可选搜索
```

**建议2: 增加隐式意图识别**

```python
# 当前: 只识别显式关键词
elif 包含("bug", "错误", "调试"):

# 建议: 增加隐式错误识别
elif 包含("bug", "错误", "调试", "崩溃", "不动", "失败", "回报"):
    → DEBUGGING
```

### 3.2 Subagent派生优化

**问题**: 当前只有5个子智能体，但某些场景可能需要更多专业化

**建议增加**:
| 子智能体 | 职责 | 触发场景 |
|----------|------|----------|
| security_expert | 安全审查 | 涉及认证、权限、加密 |
| performance_expert | 性能优化 | 涉及性能瓶颈、延迟 |
| database_expert | 数据库设计 | 涉及数据建模、SQL |

### 3.3 并行决策优化

**问题**: 当前并行决策基于任务类型，但缺少动态调整

**建议增加自适应并行度**:

```python
# 根据任务复杂度动态决定并行度
def calculate_parallel_degree(task_complexity, available_context):
    base_degree = 1
    if task_complexity == "high" and available_context > 0.7:
        base_degree = min(3, available_context * 5)
    elif task_complexity == "medium":
        base_degree = 2
    return base_degree
```

### 3.4 阶段切换优化

**问题**: DEBUGGING→EXECUTING 的切换有时不够流畅

**建议**: 增加明确的切换信号

```python
# 当前: 隐式切换
if 用户说"调试后修复":
    → DEBUGGING → EXECUTING

# 建议: 明确阶段边界
DEBUGGING:
    - 完成根因定位后
    - 如果用户说"帮我修复" → EXECUTING
    - 产出: 根因分析报告
```
