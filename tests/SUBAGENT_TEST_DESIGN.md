# Subagent集成与运行质量测试设计文档

## 概述

本文档设计针对 `agentic-workflow` skill v4.0版本的系统性测试，重点验证：
1. **各阶段触发情况** - 架构调整后能否自动匹配激活各个阶段
2. **Subagent派生与执行情况** - 各agent能否被正确派生并执行任务
3. **运行质量提升** - 执行效果、执行速度、上下文长度优化

本测试设计参考AI agent评估最佳实践：
- 来自Tavily搜索结果的行业标准
- 来自Claude Code官方文档的subagent测试方法
- 来自arXiv论文的LLM agent评估框架

## 测试执行方法

### A/B对比测试设计

```
基准组 (Baseline):
  - 不激活agentic-workflow
  - 直接执行用户请求
  - 记录: 质量、时间、Token

实验组 (Treatment):
  - 激活agentic-workflow
  - 完整流程执行
  - 记录: 质量、时间、Token、派生情况

对比维度:
  - 输出质量 (质量评分)
  - 执行时间 (毫秒)
  - Token消耗 (输入+输出)
  - 派生情况 (派生次数、并行度)
```

### 测量方法

1. **Token统计**: 使用Claude Code API的token统计功能
2. **执行时间**: 记录每个测试的开始/结束时间戳
3. **质量评分**: 使用LLM-as-a-judge方法
4. **派生追踪**: 记录Agent工具调用日志

### 测试运行命令

```bash
# 运行所有测试
python tests/run_subagent_test.py

# 查看测试结果
cat subagent_test_results.json

# 生成HTML报告
python tests/generate_subagent_report.py --input subagent_test_results.json --output subagent_report.html
```

## 测试设计原则

### 来自AI科学家视角的思考

1. **可重复性**: 每个测试必须能够独立运行并产生一致结果
2. **可测量性**: 所有指标必须有明确的计算方法
3. **全面性**: 覆盖正常路径、边界情况、错误处理
4. **独立性**: 测试之间不应该有依赖关系
5. **自动化**: 尽可能自动化执行和评估

### 验证层次

根据Tavily搜索结果中的行业最佳实践：

| 验证层 | 方法 | 适用场景 |
|--------|------|----------|
| 断言验证 | 硬性指标检查 | 触发准确率、派生成功率 |
| LLM评判 | 质量评分 | 代码质量、审查深度 |
| 性能测量 | 时间/Token统计 | 执行效率、上下文优化 |
| 日志分析 | Agent调用追踪 | 派生情况、阶段切换 |
