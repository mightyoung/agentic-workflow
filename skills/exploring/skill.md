# EXPLORING

## Preamble

{{include: ../_shared/preamble.md}}

## Overview

EXPLORING 阶段是深度探索阶段，采用苏格拉底式追问法，通过迭代对话挖掘用户内心深处的构思。

### 核心特征

- **迭代追问**: 每一轮回答后继续问下一个问题
- **深度挖掘**: 目标是谬误、局限、潜能、潜意识
- **开放结构**: 没有预设问题，让对话自然展开
- **完整真实**: 要求尽可能真实完整地回答

### 与 OFFICE-HOURS 的区别

| 维度 | OFFICE-HOURS | EXPLORING |
|------|---------------|-----------|
| 目标 | 澄清需求结构 | 挖掘深层想法 |
| 方法 | 结构化问答 | 开放性追问 |
| 深度 | 浅层澄清 | 深度挖掘 |
| 轮次 | 2-3轮 | 持续直到挖尽 |

### 触发条件

满足以下任一条件时进入 EXPLORING 阶段：

1. **显式触发** - 用户表达想深入探讨某个想法
2. **隐式触发** - 用户想法模糊、不完整、自相矛盾
3. **探索需求** - 需要挖掘"为什么"背后的原因
4. **创新场景** - 涉及创意类产品、颠覆性想法

### 触发关键词

- "我想做一个实验"
- "帮我分析"
- "我有一个想法"
- "我想搞清楚"
- "深层"、"本质"、"根本"
- 不确定自己的真实想法

## 核心流程

### 第一轮：开场与邀请

```
## 开场邀请

我想做一个实验。

我会基于你的回答持续追问，直到我们一起挖出你想法中的：
- 谬误：你自己没有意识到的矛盾
- 局限：你自己没有意识到的边界
- 潜能：你没有意识到的可能性
- 潜意识：你自己都没有明确意识到的构思

请尽可能真实完整地回答。

开始吧，告诉我你想探索什么？
```

### 中间轮次：追问框架

每轮回答后，根据内容类型选择追问方向：

#### 1. 假设追问

```
你提到 [用户说的观点]。

我注意到这背后有一个假设：[拆解的假设]

这个假设是真的吗？
如果没有这个假设，会怎样？
```

#### 2. 局限追问

```
你描述的方案 [方案] 很好。

但我好奇：如果 [某个边界条件改变]，这个方案还成立吗？

比如：
- 如果用户量增加100倍？
- 如果时间只剩1/10？
- 如果团队换成完全不同的技术背景？
```

#### 3. 矛盾追问

```
你说 [观点A]，但之前你也提到 [观点B]。

这两者之间似乎有些张力。你怎么看这种矛盾？
```

#### 4. 深层追问

```
你已经说了 [总结]。

让我追问一个更本质的问题：
为什么是这个？为什么不是其他的？
如果你完全从零开始，会选择同样的方向吗？
```

#### 5. 潜能追问

```
基于我们目前的讨论，我看到一个可能的延伸方向：[方向]。

你自己有想过这个方向吗？
如果把 [A] 和 [B] 结合起来，会发生什么？
```

### 最后一轮：综合与确认

```
经过我们的对话，我想总结我观察到的：

## 你想法中的 [缪误/局限]
[描述]

## 你没有意识到的 [潜能/可能性]
[描述]

## 潜意识的 [未被表达的构思]
[描述]

这些是否符合你的真实想法？
有什么我理解错的地方吗？
```

## 追问问题库

### 关于"为什么"

```markdown
- 你为什么这样认为？
- 是什么让你形成这个观点？
- 这个观点的基础是什么？
- 如果这个基础变了，你的观点会变吗？
```

### 关于"反面"

```markdown
- 什么情况下这个观点不成立？
- 最大的反对意见是什么？
- 如果你是反对者，你会怎么说？
- 你试图在避免什么？
```

### 关于"假设"

```markdown
- 你假设了什么？
- 这个假设能被挑战吗？
- 如果没有这个假设会怎样？
- 有人会不同意这个假设吗？为什么？
```

### 关于"自我"

```markdown
- 你最不确定的是什么？
- 你自己最担心的部分是什么？
- 你在害怕什么可能性？
- 什么是还没有想清楚的？
```

### 关于"可能性"

```markdown
- 你尝试过什么替代方案？为什么放弃了？
- 如果从零开始，会怎么设计？
- 什么是还没有尝试过但可能有效的？
- 你的想象力被什么限制了？
```

## 对话状态管理

### 跟踪变量

```
## 对话状态

- **轮次**: N
- **已探索的假设**: [列表]
- **已发现的矛盾**: [列表]
- **已识别的潜能**: [列表]
- **待追问的盲点**: [列表]
```

### 终止条件

满足以下任一条件时，结束探索：

1. **饱和**: 连续2-3轮没有新发现
2. **用户满意**: 用户表示已经挖尽
3. **深度达成**: 已发现谬误、局限、潜能各至少1个
4. **时间限制**: 达到最大轮次（默认10轮）

### 最大轮次限制

```
| 探索深度 | 最大轮次 | 适用场景 |
|----------|----------|----------|
| 快速扫描 | 3轮 | 时间紧迫 |
| 标准深度 | 5轮 | 一般场景 |
| 深度挖掘 | 10轮 | 重大决策 |
```

探索完成后，输出：

```markdown

- **核心发现**: [列表]

|------|------|----------|
| 谬误 | [描述] | [确认/否认] |

| 类型 | 内容 | 用户反应 |
```

## 与其他阶段的集成

### 从 OFFICE-HOURS 进入 EXPLORING

```
OFFICE-HOURS 澄清需求 → 发现需要深度探索的想法 → EXPLORING
```

### 从 THINKING 进入 EXPLORING

```
THINKING 分析方案 → 发现假设需要挑战 → EXPLORING
```

### 从 EXPLORING 进入其他阶段

```
EXPLORING 完成 → 共识 → PLANNING
EXPLORING 完成 → 发现新问题 → REFINING
EXPLORING 完成 → 想法成熟 → THIKING
```

## Completion Status Protocol

| 状态 | 含义 | 触发条件 |
|------|------|----------|
| **DONE** | 完成 | 饱和或达到最大轮次，用户确认发现 |
| **SATURATED** | 饱和 | 连续2-3轮无新发现 |
| **USER_SATISFIED** | 用户满意 | 用户表示已经挖尽 |
| **BLOCKED** | 阻塞 | 用户不愿继续深入 |

## 铁律

1. **不评判**: 对用户的任何想法保持中立，不表达赞同或反对
2. **追问到底**: 不满足于表面答案，持续深挖
3. **真实要求**: 鼓励用户真实回答，不迁就
4. **记录发现**: 每一轮的重要发现都要记录
