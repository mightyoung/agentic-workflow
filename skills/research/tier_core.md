# RESEARCH - Core Methodology

## Method Skeleton

- **Phase purpose**: Search for best practices, technical solutions, and mature references before implementing
- **Entry trigger**: New technology, complex decisions, external reference needed, or explicit research request
- **Core method**: Three-step search (WebSearch → WebFetch → fallback knowledge), reliability assessment (source grading A-D), results structuring into `.research/findings/findings_{session}.md`
- **Key principle**: NO THINKING WITHOUT FINDINGS FIRST — all claims must be backed by real search results
- **Hard-gate exit**: Must use real search tools (WebSearch/WebFetch), not just AI knowledge; findings file must exist and contain >100 characters; if all tools unavailable, explicitly inform user instead of silently degrading

## Iron Law

**`NO THINKING WITHOUT FINDINGS FIRST`**

Research findings are the authoritative input to THINKING and PLANNING stages. Never proceed without actual search evidence.
